## Content Standards (Every Post Must Meet All Criteria)

### E-E-A-T Requirements
- **Experience:** Include first-person operational details ("our drivers," "our fleet," "we've handled")
- **Expertise:** Reference TDLR licensing, certifications, years in service (since 2015)
- **Authoritativeness:** Cite Texas Transportation Code, TDLR regulations, AAA/NHTSA data where relevant
- **Trustworthiness:** Include real phone number CTA, no fabricated reviews, accurate service descriptions

### SEO Requirements
- **Title tag:** 50–60 characters, primary keyword near front, city name included
- **Meta description:** 140–155 characters, includes primary keyword + CTA ("Call 24/7")
- **H1:** Matches or closely mirrors title tag
- **H2s:** Target secondary keywords and question-based searches
- **Word count:** 1,200–2,000 words (comprehensive but not padded)
- **Canonical URL:** Always set to `https://texastows.com/blog/{slug}`
- **Schema:** BlogPosting schema auto-generated by BlogPostLayout component

### Localization Requirements
- Mention at least 2 specific Dallas neighborhoods, highways, or landmarks
- Reference Texas-specific laws, weather patterns, or road conditions
- Include at least one DFW-specific statistic or fact
- Use "Dallas," "DFW," or specific neighborhood names naturally throughout

### Internal Linking Requirements (minimum 3 per post)
- Link to 1–2 relevant service pages from `/services/`
- Link to 1 related blog post (once 10+ posts exist)
- Include phone number CTA link (`tel:+12145551234`) at least once
- Optional: link to service area page if neighborhood-specific

### Featured Image Requirements
- Generated via AI image generation (not stock photos)
- Dimensions: landscape, 16:9 ratio
- Must depict: tow truck, roadside scene, or Dallas-specific visual
- Alt text: descriptive, includes primary keyword and "Dallas" or "Texas"
- Hosted on CDN (CloudFront), not in repo

### No-Go List
- No pricing, rates, or cost comparisons
- No fabricated testimonials or reviews
- No competitor mentions by name
- No claims that can't be verified (e.g., "#1 in Dallas")

## Automation Script Usage

```bash
# Generate and publish a single blog post
node /home/ubuntu/texas-tows-next/scripts/publish-blog-post.mjs \
  --slug "towing-after-car-accident-dallas" \
  --date "2026-03-03"

# Dry run (generates files but does not push to GitHub)
node scripts/publish-blog-post.mjs --slug "..." --dry-run
```

The script:
1. Reads the topic from the pipeline above
2. Generates full blog post content using OpenAI GPT-4o
3. Generates featured image using the Manus image generation API
4. Uploads image to CDN
5. Creates the Next.js page file with all metadata
6. Updates blog index and sitemap
7. Commits and pushes to GitHub (triggers Vercel deploy)
